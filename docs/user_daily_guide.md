# User daily guide

このガイドは、毎朝LINE通知を見たあとに何をするかをまとめたものです。

## まず見る場所

LINE通知の最上段だけを見ます。

```text
ETF Rotation Daily YYYY-MM-DD

市場スコア
xx/100

🔴 DEFENSE / 🟡 WAIT / 🟢 CHECK BUY / 🟣 CHECK SELL
```

最初に見るのはランキングではありません。
今日の行動ラベルと「今日やること」です。

## ラベル別の行動

| ラベル | 今日の基本行動 |
|---|---|
| DEFENSE | 積立だけ。新規買いしない。ナンピンしない。 |
| WAIT | 何もしない。監視だけ。 |
| CHECK BUY | 買い候補を手動確認する。即買いではない。 |
| CHECK SELL | 保有ETFの利確/売却可否を確認する。 |

## DEFENSEの日

最も重要です。

やること:

- 積立だけ通常ルールで継続
- 新規買いしない
- ナンピンしない
- 上がっても飛びつかない
- 下がっても焦って買い増ししない

DEFENSEの日の価値は、利益を取りに行くことではなく、無駄な失敗を避けることです。

## WAITの日

やること:

- 何もしない
- 次の買い候補だけ確認
- 条件が来るまで待つ

WAITは「機会損失」ではありません。
条件が整うまで資金を守る行動です。

## CHECK BUYの日

やること:

- 候補ETFを確認
- 買い差、リスク、保有比率を確認
- 必要なら少額または分割で検討

CHECK BUYは買い命令ではありません。
最終判断はユーザーが行います。

## CHECK SELLの日

やること:

- 利確候補は、一部利確するか保有継続するか確認
- 売却候補は、新規買いせず保有継続可否を確認
- 参考保有はETF通知で買い増し判断しない

## コア資産の扱い

コア資産は長期運用が前提です。

- 401k 外国株式インデックス
- NISA オルカン
- VT / VTI / SPY / QQQ

積立は通常ルールを優先します。
暴落後や回復初期に分割買い候補が出る場合がありますが、DEFENSE中や条件が遠い場合は待ちます。

## サテライトETFの扱い

サテライトETFは、テーマの勢いとリスクを見て慎重に扱います。

買い増ししない例:

- 過熱
- 失速
- テーマ交代リスク高
- 市場全体がDEFENSE
- 個別株の誘惑で買いたくなっているだけ

## 参考保有の扱い

TDKやSOFIなどの参考保有は、ETFシステムの買い増し対象ではありません。

LINEに表示されても、ETF通知だけで買い増ししません。
別枠で保有継続、利確、損切りを考えます。

## 毎日の自己確認

LINE返信による自動記録は使いません。
必要な日だけ、手動で自己確認を記録します。

```powershell
.\scripts\run_workflow.ps1 -Command self-check -Status kept
.\scripts\run_workflow.ps1 -Command self-check -Status broke -Reason "SOFIを見て買いたくなった"
.\scripts\run_workflow.ps1 -Command self-check -Status pending -Reason "判断保留"
```

この記録は週次PDCAで使います。

## 迷った時の原則

1. DEFENSEなら買わない
2. WAITなら何もしない
3. CHECK BUYでも即買いしない
4. CHECK SELLは買い増しではなく保有確認
5. ETF通知で個別株を買い増ししない

このシステムの一番の役割は、毎日トレードさせることではありません。
買うべきでない日に、買わない判断を支えることです。

## 免責

これは投資助言ではありません。
売買の最終判断はユーザー自身が行います。
